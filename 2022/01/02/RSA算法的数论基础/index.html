RSA算法的数论基础

Created2022-01-02|Updated2022-01-02|密码学
|Post View:

一、参考书目

《数论与密码》-杨思熳 主编
《密码学与网络安全》-(美) Behrouz A.Forouzan 著 马振晗 贾军保译

二、概述

RSA密码体制是在1978年，由麻省理工的三位科学家Rivest,Shamir,Adleman基于素性检测和整数分解提出的第一个实用公钥密码体制，计算时取两个大素数p和q，计算n=p*q，将加密和解密建立在对模n的运算之上。而根据p,q求n并不难，但将n进行素因子分解的难度很大，这就是RSA安全性的依赖之处。

三、算法分析

①大素数p和q的选择：p,q都应较大且值不易接近

第一个是选择p,q的大小问题，如果有人想要攻击RSA算法，那么当n比较小时，他就可以在一定时间内用计算机将其素因子分解出来得到p,q，故理所当然，选的越大越好，目前比较安全的选择是512比特，即约2的512次方

当下，对于素性检测问题最好的判定算法运行时间是，其关于输入长度log n 呈超越多项式速度增长；同时对于整数因子分解问题，分解一个一般整数n的最好算法运行时间是O(exp(c(log n)^(1/3) (log log n)^(2/3)))，其关于输入长度log n 呈亚指数速度增长。当由p,q生成的n足够大时（带有多项式时间复杂度），这两个问题是难以解决的，因此比较安全。
另一个问题是p,q取值之差，我这里起初的想法是：采用暴力分解的话，若是找到p,q其中之一，就能根据这个值来枚举附近的一些质数，可能被破解。后面询问了一番打CTF的同学，这里是因为将n开根号后，就可以往两个方向寻找质数，若相差不大，则p,q都很容易被找到，同时这也是一种破解的方法。

②φ(n)=(p-1)*(q-1)：欧拉函数

欧拉函数的值为小于n的自然数中与n互质的个数，很明显，对于一个素数而言，其欧拉函数值为n-1。当n为两个大素数相乘之积时，n只有一种分解方法即n=pq，这是因为p和q都不能进一步拿出质因子，而试图分解p,q也需要较大运算量，此时不与n互质的数有两类，一是pi (1=<i<=(q-1))，二是qj (1<=j<=(p-1))，
下面先证明这两类数的集合没有交集：
假设存在相等的两个数，不妨令pi=qj，因为p与q互质，故必有：p|j或这q|i，又因为i<=q-1,j<=p-1，这两个都不可能成立，假设错误。
所以这两类数的集合没有交集，那么根据容斥原理，不与n互质的数一共有：pq-1-(p-1)-(q-1)=(p-1)(q-1)个，故φ(n)=φ(pq)=(p-1)*(q-1)=φ(p)φ(q)。

③e的选择与求公钥d

e应当随机选择在(1,φ(n))之间与φ(n)互素的数，这之后进行求逆操作，即ed≡1 mod n，此时要用到欧几里得除法，举个例子：
n=77=711,φ(n)=(7-1)(11-1)=60，我们从中选择e=13，现在求13d≡1 mod 60,根据欧几里得除法，
60=413+8；
13=18+5；
8=15+3；
5=13+2；
3=12+1；那么
1=3-2=3-（5-3）
=32-5=2（8-5）-5
=28-35=28-3（13-8）
=58-313=5（60-413）-313
=560-2313=560-（60-37）13
=3713-860
37*13≡1 mod 60；
可得d=37满足条件
如今对于e的选择，大部分时候会选择3，17，65537（2^16+1），因为3、17或65537分别只需要2或17次模乘运算，而一个随机选择的e(假设n是1024-bit)则大约需要1000次模乘运算。这种方法刻意把公钥指数选的小一点，其对应私钥指数就会很大，这么做的目的是节约公钥运算的时间。因为正常使用中都是用公钥加密，所以需要节约大部分人的时间，而极少部分人也会选用私钥解密，那么就只能少数服从多数。

④公钥、私钥及加解密

上述运算完成后，设置公钥为(n,e)，私钥为(n,d)，另外p，q都必须保密.
我们对明文x的选择，要求其长度大于1而小于117字节，这是因为在把明文送给RSA加密器前，要确认这个值是不是大于n，也就是如果接近n位长，那么需要先padding再分段加密，在RSA中padding也参与加密，128最大字节数减去padding的11字节就是117个字节。
下面对上述的例子进行加解密检验：
选择明文x 为 5，它满足条件x在(1,n)之间（不考虑padding的情况下），则对其加密的过程为：
C=x^e mod n=5^13 mod 77≡26 mod 77
这时要用到模重复平方算法：
设m=77,b=13,a=1,13=2^3+2^2+2^0，写成二进制形式，若该二进制形式中有2^i这一项，则n[i]=1，反之n[i]=0
n[0]=1,a0=ab≡13,b1=b^2≡169 mod 77≡15 mod 77
n[1]=0,a1=a0b1≡195 mod 77≡41 mod 77,b2=b1^2≡225 mod 77≡-6 mod 77
以此类推，最终得到a3≡26 mod 77即为所得结果。
任何知道公钥的人都可以对符合规范的明文进行加密，但解密过程只能由私钥保存者进行，可以看出，得到的C最大值为n-1，必然满足规定。
P=C^d mod n=26^37 mod 77≡5 mod 77=x
这样解密得到的明文和原始明文一样，下面将对该过程进行证明

⑤加密/解密互逆的证明

由ed≡1 mod φ(n),故ed = tφ(n)+1
现对整数段明文x分情况讨论
(a)若（x,n）=1，则有：
(x^e)^d=x^(tφ(n)+1) mod n
=(x^φ(n))^t * x mod n
=(1^t) * x mod n (欧拉定理)
=x mod n
(b)若(x,n)≠1，不妨设p|x,q |/ x，则有：
(x^e)^d = 0 mod p
(x^e)^d = x mod q
(x^e)^d = x mod p*q = x mod n
x^e mod n后为密文，再d次方之后为解密过程，原始明文=解密后得到的明文，故得证

四、算法安全性的数论分析

①因数分解攻击

当下RSA基于的想法就是在模n足够大的情况下，在适当的时间内将其分解是不可能的，或者说即使最终分解出来了，但是这一次的信息传输因为时间过去太久而失效或者过期，相当于徒劳无功。
目前要求n必须大于300个十进制数位，那么模最小就是1024比特，刚刚在对p,q的选择讨论中已经提到过，如今的因数分解算法还没有能分解带有多项式时间复杂度的大整数的，就算用世界上最快的计算机，还要进行素性检测等操作，其花费的时间同样也不可想象。

②选择密文攻击

假设私钥保存者能为其他人的任意密文解密，而攻击者拦截到了密文C (=P^e mod n),随后随机选择一个整数X，计算Y=C*(X^e) mod n，再将Y发送给私钥保存者，经解密操作后得到Z=Y^d mod n，那么攻击者就可以根据如下过程得到明文P：
Z=Y^d mod n=(C^d)(X^(ed)) mod n ≡(C^d)X mod n ，
这里是根据欧拉定理：
a^(kφ(n)+1) ≡ a mod n
ed = kφ(n)+1
继续证明：
(C^d)X mod n=(PX) mod n
故P=ZX^(-1) mod n,运用欧几里得除法求X的乘法逆就可以求得P
选择密文攻击较难实现，一种情形是密码分析者临时获得解密机器的访问权，但解密密钥被安全嵌入在设备中无法得到，此时可以通过解密大量选择的密文，利用产生的明文来推测密钥。（百度百科）

③广播攻击（低加密指数）

在选取e=3时，如果明文P过小以至于x^3<n，发送者使用相同的e进行了3次发送
C1=P^3 mod n1,C2=P^3 mod n2,C3=P^3 mod n3
运用中国剩余定理，可以得到
C’= P^3 mod n1n2n3,之后开三次方就可以求得明文P

④共模攻击

如果使用了相同的模n对相同的明文m进行了加密，且已知各条密文，那么就可以在不分解n的情况下还原出明文m的值。(e1和e2互质)

此时有：
e1s1+e2s2 = 1
式中s1、s2为一正一负的整数
通过扩展欧几里德算法，我们可以得到该式子的一组解（s1,s2），假设s1为正数,s2为负数.
因为
c1 = m^e1 mod n c2 = m^e2 mod n
所以
(c1^s1c2^s2) mod n = ((m^e1)^s1(m^e2)^s2) mod n
即
(c1^s1c2^s2) mod n = (m^(e1^s1+e2^s2)) mod n
又因
e1s1+e2s2 = 1
所以
(c1^s1c2^s2) mod n = (m^(1)) mod n
即
c1^s1*c2^s2 ≡ m mod n
得到明文m

⑤私钥d泄露后果

易知ed ≡ 1 mod φ(n)，那么一旦d泄露，相当于暴露了φ(n)，而n是公钥的一部分，那么就可以跟据n-φ(n)=pq-(p-1)*(q-1)=p+q-1，相当于把破解过程转化为加减法，这时就很容易被破解了。
当d泄露时，私钥管理者应该重新生成p,q,n,e以保证在传输信息的有效期内不会被破解，这样保证了信息的安全性。

五、个人感悟

1.RSA算法在加解密和签名验签两个方向上，都是互逆的运算过程，这其中的数论基础主要是欧拉定理
2.在使用RSA的过程中，φ(n)、d、p、q中任意一个甚至多个泄露出来都会有很大风险，因此如今在用到RSA的地方，公钥私钥刷新的速度都较快，毕竟加密总比解密简单，就算最后被解密出来了，只要消息过期了就没有用，所以在没有出现更高速有效的解密方法之前，RSA算法都是安全的
3.同上，RSA安全性依赖的是对大整数质因子分解的困难性，一旦n在有效时间内无法被分解出来，那么它就是绝对安全的加密算法
