PixieFail漏洞分析

Created2024-05-23|Updated2026-03-17
|Post View:

PXE Boot

PXE 是一种标准化客户端-服务器解决方案的规范，允许计算机通过网络启动，俗称网络启动或 Pixie 启动

除了 IPv4 之外，PXE 最初还依赖一些简单的协议来实现网络启动：DHCP、UDP 和 TFTP。这些协议被认为是相对简单的协议，可以用足够小的内存空间来实现，以适合选项 ROM 或网络接口卡 (NIC) 固件。自 2010 年发布 UEFI 规范 2.2 版以来，基于 IPv6 的 PXE 也成为了 UEFI 规范的一部分。对于 IPv6，除了核心协议之外，还实现了其他协议，例如 ICMPv6、邻居发现 (ND)、多播侦听器发现 (MLD)、UDP、TCP、DHCPv6、DNS 和 TFTP，这增加了整体攻击面

EDK II UEFI 参考实现提供了基于 IPv4 和 IPv6 的 PXE。在 EDKII 中，NetworkPkg 软件包含一个 TCP/IP 堆栈，可在初始预启动执行环境 (PXE) 阶段启用关键的网络功能。启用 PXE 环境后，机器可以通过网络连接启动，无需物理交互或键盘访问。PXE 通常用于大型数据中心，对于自动化早期启动阶段至关重要，尤其是在高性能计算 (HPC) 环境中。

通常，客户端计算机根据以下过程使用 PXE 协议从网络启动。 它涉及三方、DHCP 服务器、PXE 服务器和客户端：

  1. 客户端计算机广播一个 DHCP 数据包，该数据包请求 DHCP 和 PXE 服务器的地址。
  2. DHCP 服务器响应，发送一个广播数据包，告知客户端它是地址服务器。
  3. PXE 服务器响应客户端并报告它是启动服务器。
  4. 客户端向 DHCP 服务器发送请求，请求 IP 地址。
  5. DHCP 服务器将 IP 地址发送到客户端。
  6. 客户端将请求发送到 PXE 服务器，以请求网络启动计划（NBP）的路径。
  7. PXE 服务器响应，发送 NBP 路径。
  8. 客户端下载并运行 NBP。

Pixie Fail

PixieFail 系列的9个漏洞会影响 PXE 环境中的网络启动过程，而该过程对于在启动时从网络加载操作系统至关重要。所有发现的漏洞均存在于 DHCPv6 功能中，因为 PXE 在启动时依赖 DHCP 获取 IP 地址并发现要加载的操作系统映像的位置。

PixieFail 包括整数下溢、缓冲区溢出、越界读取、无限循环以及使用弱伪随机数生成器。这些漏洞可能导致RCE、DoS 攻击、DNS 缓存中毒以及敏感数据泄露。

CVE-2023-45237：使用弱伪随机数生成器

原漏洞

EDK II 的IP堆栈中使用了伪随机数生成器，在AMI Aptio中其定义如下 MdeModulePkg\Include\Library\NetLib.h：

在整个 NetworkPkg 堆栈中，上述宏通常从 MdeModulePkg\Library\DxeNetLib\DxeNetLib.c 中定义的函数Seed中获取值，如下所示：NetRandomInitSeed()

漏洞描述：

上述函数根据调用该函数的平台时钟时间和平台的单调计数器输出一个整数值。该 C 语言惯用法NET_RANDOM (NetRandomInitSeed()在多个NetworkPkg函数中用于为各种网络协议字段生成据称随机的唯一数字标识符。以这种方式生成数字不会产生具有均匀分布的随机数，而是会产生易于预测的数字和有偏差的分布。此外，在不同的网络层中使用相同的惯用法为协议的字段或对象分配所谓的随机值，使得外部观察者能够通过获取任何此类网络层中使用的数字样本来恢复生成器的内部状态。

关于gBS->GetNextMonotonicCount：

这个值对于同一个机器而言是可预测的

因此微星自己改的方案就是：把时间、系统guid、MonotonicCount合到一个结构体里面然后算CRC32，如果固件不支持CRC32就用AMI默认提供的方案，即去掉系统guid（只有这个无法预测）

借助DMA的思路

绕过微星修复方案的方法就是将gBS->CalculateCrc32的函数地址替换为另一个三参数的gBS函数，然后返回值使得EFI_ERROR为true

CVE-2023-45233：解析目标选项标头中的 PadN 选项时出现无限循环

函数Ip6IsOptionValidin会被调用来验证IPv6 数据包头中的目标选项头，其中一种受支持的类型是 PadN。在这种情况下，通过将PadN 选项的Length字段的值加2来更新Offset变量

如果选项的长度字段为0xFE，则0x100（0xFE+2）将被更新为Offset的值，但在这里被截断为UINT8，这意味着当将0x100添加到Offset时，它将变成0x0，因此会出现无限循环
